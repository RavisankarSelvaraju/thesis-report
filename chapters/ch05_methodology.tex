    \chapter{Methodology} \label{chap:methodology}
    
    This chapter provides details on the proposed approach to improve the odometry estimation in the Coyote~3 rover system. The details of \gls{inekf}-based odometry, the proposed measurement update step for the \gls{inekf}, the Mahalanobis-distance-based measurement rejection, the contact spoke estimation, and an overview of the proposed approach are presented in this chapter.

    The proposed approach uses \gls{imu} data to propagate the filter state and joint encoder data to construct the measured \gls{pcv}. This section explains the intuition behind the proposed approach. The predicted \gls{pcv} vector is calculated for every spoke of each wheel. To simplify the explanation, only one wheel and its spokes are considered here. Figure~\ref{fig:proposedApproachOverviewFlowDiagram} presents a flow diagram, providing an outline of the proposed approach. The simplified calculation blocks inside the \gls{inekf}, the data flow from sensors and kinematics into the \gls{inekf} and the inconsistent wheel encoder measurement rejection approach.
    
    In this thesis, the \gls{inekf} estimates the pose and velocity of the rover body with respect to the odometry frame, denoted as $odom$. This frame is initialized at the start of the estimator component and remains fixed during the estimation. The \gls{inekf} estimates the pose of the rover and other elements in its state vector. To estimate these elements, the filter uses the \gls{imu} data to propagate the state vector elements forward in time. Let the propagated state be $\hat{x}^{-}_{k+1}$ at the time step $k+1$. Let the current state be $x_{k}$ and the current time step be $k$. This propagation of state is done using an inertial motion model driven by the \gls{imu} measurement. This motion model uses the bias-corrected gyroscope and accelerometer values to estimate the orientation, position and velocity of the rover at $k+1$. Since this predicted state is calculated only with \gls{imu} driven motion model, it accumulates error due to sensor drift. This error needs to be corrected by an independent measurement source. In the proposed approach, the correction is done using the joint encoder measurements.
    
    Figure~\ref{fig:approachPCVexplanation} shows the front left wheel of the Coyote~3 rover in the body frame coordinate system. The wheel is rotating in the clockwise direction, resulting in the rover moving forward. The purpose of this figure is to show the predicted and measured \gls{pcv} vectors expressed in the body frame. 
    
    Using the predicted state $\hat{X}^{-}_{k+1}$, the velocity of each spoke tip can be calculated in the body frame. The details of the predicted \gls{pcv} are provided in Section~\ref{subsec:predictedPCV}. The velocity of each spoke tip is calculated using the predicted body linear velocity, the bias-corrected body angular velocity from the \gls{imu}, and the spoke-tip position relative to the body frame. This calculation generates a velocity vector for each spoke tip of the selected wheel, shown as red arrows in Figure~\ref{fig:approachPCVexplanation}. 
    
    To correct the predicted state $\hat{x}^{-}_{k+1}$, the measured \gls{pcv} is calculated using the joint encoder measurements. The details of the measured \gls{pcv} are provided in Section~\ref{subsec:measuredPCV}. Similar to the predicted \gls{pcv}, we calculate the measured \gls{pcv} for all the spokes of a wheel. The measured \gls{pcv} is calculated from the wheel angular velocity about its rotation axis and the position vector from the wheel center to the spoke tip. The resulting measured \gls{pcv} points along the direction of wheel rotation and is perpendicular to the position vector from the wheel center to the spoke tip. These vectors are shown as the green arrow in Figure~\ref{fig:approachPCVexplanation}. This measured \gls{pcv} is initially expressed in the wheel frame and later transformed into the body frame.
        
    Once both the predicted and measured \gls{pcv} are calculated, the spokes assumed to be in contact need to be selected and used to update the state vector. This can be done in two ways, as explained in Section~\ref{sec:ContactEstimation}. The proposed approach uses the \gls{lf} method to select the spokes that are used in the filter update step. These selected spokes are used to calculate the residual as discussed in Section~\ref{subsec:residual}. After the residual is calculated, the uncertainty of the residual is represented by the innovation covariance~$S$, calculated using Equation~\ref{eq:innovationCovariance}. This covariance describes how much variation is expected between the predicted and measured \gls{pcv} vectors considering the uncertainty in the filter state and the measurement noise. The squared Mahalanobis distance is calculated using the innovation covariance. This distance represents how consistent the measured \gls{pcv} is with the filter's predicted \gls{pcv}, relative to the expected uncertainty of the residual.

    From this information, it can be statistically decided whether a spoke measurement is consistent with the predicted state. If the squared \gls{md} is above the chi-squared threshold, the corresponding spoke measurement is considered inconsistent, which may indicate wheel slip or incorrect contact assumption and is rejected. This spoke is rejected from the update step. Only spokes that are selected as in contact and with squared \gls{md} within the threshold are used to update the filter state. After the update, the filter produces the corrected state $\hat{x}^{+}_{k+1}$. 
    
    The calculated \gls{md} can also be used to estimate the spokes that are likely to be in contact. The spoke with the least \gls{md} is selected as the most likely to be in contact. This can be seen in Figure~\ref{fig:approachPCVexplanation}, where the spoke estimated to be in contact with the ground has the predicted and measured \gls{pcv} vectors that are consistent with each other. The contact spoke is therefore expected to have a lower residual and \gls{md} value than the other spokes. The contact estimation is explained in detail in Section~\ref{subsec:mahala-based-contact-estimation}. 
    
    In the implemented filter update, the \gls{lf} method is used to select the contact spokes. The \gls{md} is then used to reject inconsistent spoke measurements before the update. In addition, the \gls{md} values-based contact estimation is a possible alternative to the \gls{lf} method, as explained in Section~\ref{subsec:mahala-based-contact-estimation}.
    
    \section{Invariant EKF-based Odometry}
    
    The \gls{inekf} is an extension of the Extended Kalman Filter whose state evolves on Lie groups. In the \gls{inekf}, the estimation error is defined as an invariant error formulation, such as left or right invariant error rather than Euclidean vector subtraction. The details on the \gls{inekf} and the invariant error are explained in Chapter~\ref{chap:background}. The proposed approach for odometry estimation in \gls{rwr} uses the proprioceptive sensors available on the rover. The \gls{imu} and the joint encoders are used in the fusion process. The filter's state vector and the details on the propagation step are provided in the following subsections.

    \subsection{Filter State Vector}    
    The state vector $X$ contains the quantities that need to be estimated by the filter. In this work, the state vector contains position, orientation, velocity and \gls{imu} sensor biases. These quantities are required to estimate the rover motion. The filter state is constructed as shown in Equation~\ref{eq:stateVector}. 
    
    \begin{equation}
        x = [R, v, p, b_g, b_a]
        \label{eq:stateVector}
    \end{equation}
    where $R$ is the rotation matrix that represents the orientation of the body frame with respect to the $odom$ frame. $v$ is the velocity vector of the body in the $odom$ frame, $p$ is the position vector of the body in the $odom$ frame, $b_g$ is the gyroscope bias, and $b_a$ is the accelerometer bias. 
    
    \subsection{Propagation step}
    
    The propagation step propagates the current filter state defined in Equation~\ref{eq:stateVector}, to the next time step using the inertial motion model and the \gls{imu} data to generate the State vector at $k+1$ time. The state covariance values are propagated using the state transition and process noise matrices. This thesis does not modify the propagation step and uses the original implementation \cite{REPO-inekf-github}. The approach presented by Hartley et al. \cite{Hartley2018ContactAidedInEKF}, the authors of the original \gls{inekf} implementation, augments the state vector by adding the positions of contact points. In contrast, this thesis keeps the state vector unchanged and estimates only the state variables listed in Equation~\ref{eq:stateVector}.
    
    \section{Measurement Update Step}
    
    This section presents the parts of the proposed approach adapted for the \gls{rwr} system, primarily the \gls{pcv}-based measurement update and its use in the \gls{inekf} correction step. For the fusion process, the \gls{pcv} vectors are calculated. This is the velocity of a wheel spoke tip expressed in the body frame in a \gls{rwr} system. The \gls{pcv} can also be referred to as spoke-tip velocity or contact velocity. The term "Potential" refers to the possibility that any one of the spokes in a wheel can come into contact, depending on the terrain. The \gls{pcv} is calculated in two forms. The \gls{pcv} vector, calculated from the filter state vector, after the prediction step of the \gls{inekf} is called predicted \gls{pcv}. In the update step, the \gls{pcv} vector is calculated from the joint encoder measurements called measured \gls{pcv}. Within the \gls{inekf}, the predicted and measured \gls{pcv} vectors are used to calculate the residual, also known as the innovation.
    
    \subsection{Predicted Potential Contact Velocity} \label{subsec:predictedPCV}
    
    The predicted \gls{pcv} is calculated after the propagation step of the \gls{inekf}. The vector quantities used in the predicted \gls{pcv} calculation are expressed in the body frame of the rover. This is done to ensure the consistency between predicted and measured \gls{pcv}. The velocity of a \gls{pc} fixed to the rover can be calculated with Equation~\ref{eq:velocityOfaPointInRigidBody}. The rigid body equation is negated to obtain the predicted \gls{pcv} as shown in Equation~\ref{eq:measurementFunction}.
        
    \begin{equation}
        v_{pc,rigid}^{body} = ([R_{body}^{odom}]^T  v_{body}^{odom} + (\omega_{body}^{body} \times r_{pc}^{body}))
        \label{eq:velocityOfaPointInRigidBody}
    \end{equation}
    
    \begin{equation}
        v_{pc, predicted}^{body} = -([R_{body}^{odom}]^T  v_{body}^{odom} + (\omega_{body}^{body} \times r_{pc}^{body}))
        \label{eq:measurementFunction}
    \end{equation}
    
    \begin{equation}
        \omega_{body}^{body}
        =
        \omega_{meas}^{body}
        -
        b_g.
        \label{eq:gyroBiasRemoved}
    \end{equation}
    
    where, $R_{body}^{odom}$ is the orientation of the body frame expressed in the $odom$ frame and $[R_{body}^{odom}]^T$ transforms the $v_{body}^{odom}$ to the $v_{body}^{body}$. The $R_{body}^{odom}$ and the $v_{body}^{odom}$ are part of the filter's state vector shown in Equation~\ref{eq:stateVector}. $v_{body}^{odom}$ represents the linear velocity of the body frame expressed in the $odom$ frame. $\omega_{body}^{body}$ is the bias-corrected angular velocity of the body expressed in the body frame. $r_{pc}^{body}$ is the position vector of the \gls{pc} from the body frame of the rover expressed in the body frame. The $v_{pc, rigid}^{body}$ is the rigid-body velocity of the potential contact point at the spoke tip, expressed in the body frame. It represents how fast the spoke tip would move if it were rigidly fixed to the rover body and the wheel were not rotating.
    
    This quantity is calculated in Equation~\ref{eq:velocityOfaPointInRigidBody}. The negated form represents the relative velocity of the contact surface with respect to the rover body at this potential contact point. This quantity is used as the predicted \gls{pcv}, or measurement function $h(x)$, as shown in Equation~\ref{eq:measurementFunction}.

    This can be understood using the difference between an ice skate and a roller skate. A point on an ice skate blade is fixed relative to the boot, so its velocity is caused only by the translation and rotation of the skater. This is analogous to $v_{pc, rigid}^{body}$, which describes the motion of the spoke tip due only to the rover body motion. In contrast, a point on a roller-skate wheel also has an additional velocity component caused by the wheel rotation.
    
    To understand the negation in Equation\ref{eq:measurementFunction}, we should look at an example. Imagine we have a rover that is moving forward with a velocity of $+2 m/s$. For the sake of simplicity, our imaginary rover has no rotational velocity. According to rigid body kinematics, the velocity of a point attached to the rover will also be $+2 m/s$. From the rover's body frame of reference, the forward motion of the body will be seen as the ground moving backwards under the rover. Mathematically, this can be represented by the negation of Equation~\ref{eq:velocityOfaPointInRigidBody}, resulting in the velocity of the ground moving under the point of contact to be $-2 m/s$. This perspective is used to compare the predicted and the measured \gls{pcv}
        
    \subsection{Measured Potential Contact Velocity} \label{subsec:measuredPCV}
    
    The measured \gls{pcv} is calculated using the wheel's angular velocity expressed as a vector and the position vector of \gls{pc}. The position vector is calculated from the wheel center to the tip of the spoke. The measured \gls{pcv} vector is perpendicular to the position vector and points in the direction of rotation of the wheel. The calculation of the measured \gls{pcv} is shown in Equation~\ref{eq:MeasuredPCVCalculation}. The measured \gls{pcv} vector is transformed into the body frame to be consistent with the predicted \gls{pcv} prior to the residual calculation.
    
    \begin{equation}
        v_{pc, measured}^{body} = R_{wheel}^{body} (\omega_{wheel}^{wheel} \times {}^{}r_{pc}^{wheel})
        \label{eq:MeasuredPCVCalculation}
    \end{equation}

    From the rover's body frame of reference, $v_{pc, measured}^{body}$ represents the velocity of the ground moving under the point of contact. This is the same quantity as the $v_{pc, predicted}^{body}$, so the difference between them can be calculated as the residual. The $\omega_{wheel}^{wheel}$ is the angular velocity of the wheel expressed in the wheel frame. The $r_{pc}^{wheel}$ is the position vector of the \gls{pc} from the wheel center expressed in the wheel frame. $R_{wheel}^{body}$ is the orientation of the wheel frame expressed in the body frame.
      
    \subsection{Residual Calculation} \label{subsec:residual}
    
    The residual, also referred to as the innovation, is calculated as the difference between the measured and the predicted \gls{pcv}. Both quantities are expressed in the body frame of reference before comparison. The residual is used in the \gls{inekf} to update the state vector.
    
    \begin{equation}
        \boldsymbol{\nu}_{pcv} =  {v}_{pc, measured}^{body} - {v}_{pc, predicted}^{body} 
    \label{eq:pcvResidual}
    \end{equation}
    
    If the $\boldsymbol{\nu}_{pcv}$ is small, the wheel encoder measurements are consistent with the filter's predicted rover motion. If the residual is large, then the wheel encoder measurements are inconsistent with the filter's prediction. This could mean that wheel encoder measurements may be affected by errors such as wheel slip, an inconsistent contact assumption, or noise in the measurements. The residual is also used in the \gls{md} calculation, which is later used in slip rejection and contact estimation.

    \section{Measurement Model and Jacobian}
    \label{sec:measurementModelAndJacobian}

    The measurement model, also referred to as the measurement function $h(x)$, corresponds to the predicted \gls{pcv}. It is used by the \gls{inekf} to calculate a predicted \gls{pcv} from the filter state. The measurement function given in Equation~\ref{eq:measurementFunction} is rewritten as

    \begin{equation}
        h(x)
        =
        -\left(
            [R_{body}^{odom}]^{T}v_{body}^{odom}
            +
            \left(
                \omega_{body}^{body}
                \times
                r_{pc}^{body}
            \right)
        \right)
        \label{eq:measurementFunction-2}
    \end{equation}

    Where $x$ represents the complete filter state used by the measurement function. The measurement function also depends on the current gyroscope measurements. For the sake of compactness, it is not shown explicitly. The complete filter state consists of the Lie-group state $X$ and the bias states grouped as $b$, as shown in Equation~\ref{eq:State}

    \begin{equation}
        X
        =
        \left(
            R,
            v,
            p
        \right), \\ 
        b = \left(
            b_g,
            b_a
        \right)
        \label{eq:State}
    \end{equation}
    
    \begin{equation}
        x
        =
        \left(
            X,
            b
        \right).
        \label{eq:completeFilterState}
    \end{equation}
 
    The Lie-group state $X$ contains the rotation, velocity, and position, while the bias state $b$ contains the gyroscope $b_g$ and acceleration $b_a$ biases as Euclidean variables. During the correction step, the current state estimate is the propagated state $\hat{x}_{k+1}^{-}$. For compactness, this state is denoted by $\hat{x}$ in the remainder of this section:

    \begin{equation}
        \hat{x}
        =
        \hat{x}_{k+1}^{-}
        =
        \left(
            \hat{X},
            \hat{b}
        \right)
        \label{eq:estimatedCompleteFilterState}
    \end{equation}

    Where $\hat{X}$ is the propagated estimate of the Lie-group state, and $\hat{b}$ is the propagated bias estimate. The predicted measurement at the current state estimate is therefore

    \begin{equation}
        h(\hat{x})
        =
        -\left(
            [\hat{R}_{body}^{odom}]^{T}
            \hat{v}_{body}^{odom}
            +
            \left[
                \left(
                    \omega_{meas}^{body}
                    -
                    \hat{b}_g
                \right)
                \times
                r_{pc}^{body}
            \right]
        \right).
        \label{eq:predictedMeasurementAtEstimate}
    \end{equation}

    The measurement Jacobian describes how the predicted \gls{pcv} changes under a small perturbation to the current state estimate. The local perturbation vector $\xi$ is defined as

    \begin{equation}
        \xi
        =
        \begin{bmatrix}
            \xi^R &  \xi^v & \xi^p & \zeta^{b_g} & \zeta^{b_a}
        \end{bmatrix} ^{T},
        \label{eq:ErrorStateVector}
    \end{equation}

    Where, $\xi^R$ is the local rotation perturbation, $\xi^v$ is the velocity perturbation, $\xi^p$ is the position perturbation, $\zeta^{b_g}$ is the gyroscope bias perturbation, and $\zeta^{b_a}$ is the accelerometer bias perturbation. The first three elements of $\xi$ are associated with the Lie-group state and are grouped as
    
    \begin{equation}
        \xi^X
        =
        \begin{bmatrix}
            \xi^R \\
            \xi^v \\
            \xi^p
        \end{bmatrix}.
        \label{eq:LieGroupErrorState}
    \end{equation}

    \begin{equation}
        \xi^b
        =
        \begin{bmatrix}
            \zeta^{b_g} \\
            \zeta^{b_a} 
        \end{bmatrix}.
        \label{eq:BiasesErrorState}
    \end{equation}
    
    For the Jacobian calculation, a state close to the current estimate is defined by applying this increment through right multiplication:

    \begin{equation}
        X_{pert}
        =
        \hat{X}
        \operatorname{Exp}(\xi^X),
        \label{eq:perturbedLieGroupState}
    \end{equation}

    Where $X_{pert}$ can be referred to as the Lie group state perturbed by $\xi^{X}$, $\operatorname{Exp}(\xi^X)$ maps the local perturbation coordinates to a Lie-group increment. This increment contains the small changes associated with rotation, velocity, and position. Equation~\ref{eq:perturbedLieGroupState} does not define an additional filter state. It only defines a state close to $\hat{X}$ for the linearization of the measurement function. The gyroscope and accelerometer biases are Euclidean variables. Therefore, their perturbations are applied using vector addition:

    \begin{align}
        {b_{g, pert}}
        &=
        \hat{b}_g+ \zeta^{b_g},
        \label{eq:perturbedGyroBias}
        \\
        {b_{a, pert}}
        &=
        \hat{b}_a+\zeta^{b_a}.
        \label{eq:perturbedAccelerometerBias}
    \end{align}
        
    \begin{equation}
        b_{\mathrm{pert}}
        =
        \begin{bmatrix}
            b_{g,\mathrm{pert}} \\
            b_{a,\mathrm{pert}}
        \end{bmatrix}
        \label{eq:perturbedBiases}
    \end{equation}

    where, $b_{\mathrm{pert}}$ can be referred to as biases state perturbed by $\xi^{b}$. 
    Using Equations~\ref{eq:perturbedLieGroupState}-- \ref{eq:perturbedAccelerometerBias}, the complete perturbed state is defined as

    \begin{equation}
        x_{pert}
        =
        [
            X_{pert},
            b_{pert}
        ].
        \label{eq:perturbedState}
    \end{equation}

    The measurement Jacobian can now be defined with respect to the local perturbation vector:

    \begin{equation}
        H
        =
        \left.
        \frac{\partial h(
        x_{pert})}
            {\partial \xi}
        \right|_{\xi=0}.
        \label{eq:MeasurementJacobian-1}
    \end{equation}
    
    The derivative is evaluated at $\xi=0$ because this corresponds to linearizing the measurement function around the current propagated state estimate $\hat{x}$. The Jacobian therefore describes the first-order change in the predicted \gls{pcv} caused by small changes in the rotation, velocity, position, and bias estimates. Using the perturbation ordering defined in Equation~\ref{eq:ErrorStateVector}, the measurement Jacobian consists of the following blocks:

    \begin{equation}
        H
        =
        \left.
        \begin{bmatrix}
            \dfrac{\partial h(x_{pert})}
                {\partial\xi^{R}}
            &
            \dfrac{\partial h(x_{pert})}
                {\partial\xi^{v}}
            &
            \dfrac{\partial h(x_{pert})}
                {\partial\xi^p}
            &
            \dfrac{\partial h(x_{pert})}
                {\partial\zeta^{b_g}}
            &
            \dfrac{\partial h(x_{pert})}
                {\partial\zeta^{b_a}}
        \end{bmatrix}
        \right|_{\xi=0}.
        \label{eq:MeasurementJacobian-2}
    \end{equation}

    Evaluating these partial derivatives gives

    \begin{equation}
        H
        =
        \begin{bmatrix}
            -\left[
                [\hat{R}_{body}^{odom}]^{T}
                \hat{v}_{body}^{odom}
            \right]_{\times}
            &
            -\mathbf{I}
            &
            \mathbf{0}
            &
            -[r_{pc}^{body}]_{\times}
            &
            \mathbf{0}
        \end{bmatrix}.
        \label{eq:MeasurementJacobian-3}
    \end{equation}

    The rotation block describes how a small change in the estimated orientation changes the body-frame velocity used in the predicted \gls{pcv}. The velocity block is $-\mathbf{I}$ because the predicted \gls{pcv} directly depends on the negative body-frame velocity. The position block is zero because the measurement function does not directly depend on the rover position. The gyroscope bias block is non-zero because the bias affects the bias-corrected angular velocity in Equation~\ref{eq:gyroBiasRemoved}. The accelerometer bias block is zero because the accelerometer bias does not directly appear in the measurement function.

    The detailed evaluation of Equation~\ref{eq:MeasurementJacobian-2}, resulting in Equation~\ref{eq:MeasurementJacobian-3}, is provided in Appendix~\ref{appendix:MeasurementJacobianDerivation}.

    The validity of the measurement Jacobian is verified by comparing the analytical Jacobian with a numerical Jacobian. The numerical Jacobian is calculated by applying small perturbations to the variables in Equation~\ref{eq:ErrorStateVector} and evaluating the resulting change in the measurement function. The analytical and numerical Jacobians are found to be consistent. The numerical Jacobian calculation is provided in Appendix~\ref{appendix:NumericalJacobianCalculation}.

    \section{Contact Estimation} \label{sec:ContactEstimation}
            
        From the residual $\boldsymbol{\nu}$ calculated in Subsection~\ref{subsec:residual}, the squared \gls{md} is calculated. The squared \gls{md} is a statistical measure of the distance between the predicted and measured \gls{pcv} vectors, accounting for uncertainty in the filter state and measurement noise. The squared \gls{md} measures the residual between a measurement and its predicted values weighted by the inverse innovation covariance \cite{SITE-mathworks-mahalanobis}. The squared \gls{md} is calculated for all the spokes of the rover. It is calculated using the residual and its covariance. The residual covariance and the squared \gls{md} calculation are given by Equations~\ref{eq:innovationCovariance} and \ref{eq:mahalanobisDistanceSquared}, respectively.
        
        \begin{equation}
        \boldsymbol{\Sigma}_{\nu}
        =
        H P H^{T}
        +
        \boldsymbol{\Sigma}_{m}, 
        \label{eq:innovationCovariance}
        \end{equation}
        
        \begin{equation}
        d_M^2
        =
        \boldsymbol{\nu}^{T}
        \boldsymbol{\Sigma}_{\nu}^{-1}
        \boldsymbol{\nu}
        \label{eq:mahalanobisDistanceSquared}
        \end{equation}
        
        where $\boldsymbol{\Sigma}_{\nu}$ is the residual or innovation covariance, $H$ is the measurement Jacobian calculated in the previous section, $P$ is the state covariance estimated by the filter, $\boldsymbol{\Sigma}_{m}$ is the measurement noise covariance, and $\boldsymbol{\nu}$ is the residual.
        Once the squared \gls{md} is calculated for all spokes, it is used to estimate which spokes are in contact with the ground. In the thesis, after some initial testing, it was found that using the \gls{md} method of contact estimation makes the filter unstable. The error in the contact estimation is fed to the filter, which increases the state covariance; then that same value is used to calculate the \gls{md} in the next iteration. This leads to a feedback loop that makes the filter unstable. Therefore, the \gls{lf} method is used to estimate contact in the implemented filter. The \gls{md} method is still presented here as it is a valid approach and can be used in future work.        
        
        The rimless wheel motion can be described as a combination of the swing and contact phases. Correctly identifying the contact spoke is important because the \gls{pcv} residual is calculated for a specific spoke tip. Selecting the wrong spoke can make a valid wheel measurement appear inconsistent, leading to an incorrect measurement rejection. The contact estimation is performed using two methods, the lowest-foot method and the \gls{md}-based method. 
        
        \subsection{Lowest-Foot-based Contact Estimation}
        
        A geometric method for estimating the contact spokes is to assume that the spoke tip with the lowest vertical position is in contact. This can be estimated using robot kinematics and the wheel's joint positions. The pose of each spoke with respect to the body frame can be calculated, and the spoke in contact can be estimated. The \gls{rw} is constructed with five spokes, so geometrically there could be either one spoke in contact or two. The "\gls{lf}" name comes from the single-contact-spoke assumption. This method can also be used for multi-contact spoke estimation. For multi-contact spoke estimation, we assumed a vertical position region; if the spoke tip is within this region, it is estimated as in contact. This assumption works well on a flat, solid terrain, but on an inclined or rough terrain with obstacles, this assumption may not hold. Even though the \gls{lf} can estimate multiple contact spokes, the naming convention is kept as the "Lowest Foot" method.
              
        \subsection{Mahalanobis-Distance-based Contact Estimation} \label{subsec:mahala-based-contact-estimation}
        
        The use of \gls{inekf} for odometry estimation provides an opportunity to estimate the spokes that are in contact. With the squared \gls{md} calculated in the previous section, the contact spokes can be estimated. The squared \gls{md} value for all the spoke tips for all the wheels is calculated. Once the calculation is done, the calculated squared \gls{md} values for each wheel are sorted. From this sorted array, the spoke with the least squared \gls{md} value is selected as the spoke \texttt{in\_contact}. This is based on the assumption that the spoke tip in contact with the ground produces the smallest difference between the predicted and measured \gls{pcv}. Therefore, the spoke with the lowest squared \gls{md} value is selected as the most likely contact spoke. This method can also be used to estimate multiple contact spokes. Once the first spoke in contact is estimated, then the next smallest squared \gls{md} value is selected, and the difference between the first and second smallest squared \gls{md} values is calculated. If this difference is below a certain threshold, the second spoke is also estimated to be in contact. The implementation details related to the approach discussed in this chapter is presented in the Implementation Chapter~\ref{chap:implementation}.
        
        \section{Slip-aware Measurement Rejection} \label{sec:sliprejection}
        
        The measured \gls{pcv}, calculated from the joint encoder measurements, can be affected by wheel slip. This wheel slip can occur due to several factors such as slippery terrain, compliant terrain like soft sand, loose and uneven terrain like pebbles, inclined terrain, very high wheel speeds or a combination of these factors. The nature of slip and the details required to understand the wheel slip are discussed in Section~\ref{sec:background_wheelslip}. 
        
        In the \gls{inekf} framework, slip-induced inconsistencies may be identified when the measured \gls{pcv} vector from the joint encoders differs from the predicted \gls{pcv} computed from the filter state. This inconsistency can be computed using the squared \gls{md} from the previous section. 
                
        The squared \gls{md} values calculated for all the spokes are compared with the chi-squared threshold. The chi-squared threshold value is used because the residual is assumed to be Gaussian distributed. Therefore, the squared \gls{md} follows approximately a chi-squared distribution~\cite{barshalom2001estimation,chen2023kalman}. The degree of freedom for the chi-squared distribution is equal to the dimension of the residual. In this case, the residual is a 3D vector, so the degree of freedom is 3. The chi-squared threshold value is calculated using the chi-squared distribution at a 95\% confidence level. Hence, the threshold value is 7.815. 
        
        The spokes selected as \texttt{in\_contact} by the contact estimation method are checked for slip. If the squared \gls{md} for a spoke that is \texttt{in\_contact} is above the threshold, then it is considered to be \texttt{slipping}. For the update step, only the spokes that are \texttt{in\_contact} and \texttt{non-slipping} are used. Spokes that are \texttt{in\_contact} but \texttt{slipping} are rejected during the update step. The implementation details of the slip-aware measurement rejection are presented in Subsection~\ref{subsec:implementationSlipRejectionContactSelection}.
