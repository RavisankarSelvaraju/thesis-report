    \chapter{Methodology} \label{chap:methodology}
    
    This chapter provides details on the proposed approach to improve the odometry estimation in the coyote~3 rover system. The details of \gls{inekf}-based odometry, the proposed measurement update step for the \gls{inekf}, the Mahalanobis-distance-based measurement rejection, the contact spoke estimation, and an overview of the proposed approach are presented in this chapter. 
    
    \section{Overview of the Proposed Approach} \label{sec:approach_overview}

    The proposed approach uses \gls{imu} data to propagate the filter state and joint encoder data to construct the measured \gls{pcv}. This section explains the intuition behind the proposed approach. The predicted \gls{pcv} vector is calculated for every spoke of each wheel. To simplify the explanation, only one wheel and its spokes are considered here. Figure~\ref{fig:proposedApproachOverviewFlowDiagram} presents a flow diagram, providing an outline of the proposed approach. The simplified calculation blocks inside the \gls{inekf}, the data flow from sensors and kinematics into the \gls{inekf} and the inconsistent wheel encoder measurement rejection approach.
    
    In this thesis, the \gls{inekf} estimates the pose and velocity of the rover body with respect to the odometry frame, denoted as $odom$. This frame is initialized at the start of the estimator component and remains fixed during the estimation. The \gls{inekf} estimates the pose of the rover and other elements in its state vector. To estimate these elements, the filter uses the \gls{imu} data to propagate the state vector elements forward in time. Let us define this propagated state $\hat{X}^{-}_{k+1}$ at the time step $k+1$. Let the current state be $X_{k}$ and the current time step be $k$. This propagation of state is done using an inertial motion model driven by the \gls{imu} measurement. This motion model uses the bias corrected gyroscope and accelerometer values to estimate the orientation, position and velocity of the rover at $k+1$. Since, this predicted state is calculated only with \gls{imu} driven motion model, it accumulates error due to sensor drift. This error needs to be corrected by an independent measurement source. In the proposed approach, the correction is done using the joint encoder measurements.
    
    Figure~\ref{fig:approachPCVexplanation} presents the front left wheel of the Coyote~3 rover and the body frame coordinate system. The wheel is rotating in the clock-wise direction resulting in rover moving forward. The purpose of this figure is to show the predicted and measured \gls{pcv} vectors expressed in body frame. 
    
    Using the predicted state $\hat{X}^{-}_{k+1}$, the velocity of each spoke tip can be calculated in the body frame. The details of the predicted \gls{pcv} are provided in Section~\ref{subsec:predictedPCV}.
    To understand the predicted \gls{pcv}, we calculate the velocity that each spoke tip would have due to the predicted body linear velocity, the bias-corrected body angular velocity from \gls{imu}, and the spoke-tip position relative to the body frame. This calculation generates a velocity vector for each spoke tip of the selected wheel, shown as red arrows in Figure~\ref{fig:approachPCVexplanation}. 
    
    To correct the predicted state $\hat{X}^{-}_{k+1}$, the measured \gls{pcv} is calculated using the joint encoder measurements. The details of the measured \gls{pcv} are provided in Section~\ref{subsec:measuredPCV}. Similar to the predicted \gls{pcv}, we calculate the measured \gls{pcv} for all the spokes of a wheel. The measured \gls{pcv} is calculated from the wheel angular velocity about its rotation axis and the position vector from the wheel center to the spoke tip. The resulting measured \gls{pcv} points along the direction of wheel rotation and is perpendicular to the position vector from the wheel center to the spoke tip. These vectors are shown as the green arrow in Figure~\ref{fig:approachPCVexplanation}. This measured \gls{pcv} is initially expressed in the wheel frame and later transformed into the body frame.
        
    Once both the predicted and measured \gls{pcv} are calculated, the spokes assumed to be in contact need to be selected and used to update the state vector. This can be done in two ways as explained in Section~\ref{sec:ContactEstimation}. The proposed approach uses the \gls{lf} method to select the spokes that are used in the filter update step. These selected spokes are used to calculate the residual as discussed in Section~\ref{subsec:residual}. After the residual is calculated, the uncertainty of the residual is represented by the innovation covariance $S$, calculated using Equation~\ref{eq:innovationCovariance}. This covariance describes how much variation expected between the predicted and measured \gls{pcv} vectors considering the uncertainty in the filter state and the measurement noise. The squared Mahalanobis distance is calculated using the innovation covariance. This distance represents how consistent the measured \gls{pcv} is with the filter's predicted \gls{pcv}, relative to the expected uncertainty of the residual.

    From this information, it can be statistically decided whether a spoke measurement is consistent with the predicted state. If the squared \gls{md} is above the chi-squared threshold, the corresponding spoke measurement is considered inconsistent, which may indicate wheel slip or incorrect contact assumption and is rejected. This spoke is rejected from the update step. Only spokes, with squared \gls{md} within the threshold are used to update the filter state. After the update, the filter produces the corrected state $\hat{X}^{+}_{k+1}$. 
    
    The calculated \gls{md} can also be used to estimate the spokes that are likely to be in contact. The spoke with the least \gls{md} is selected to be most likely in contact. This can be seen in Figure~\ref{fig:approachPCVexplanation}, where the spoke estimated to be in contact with the ground has the predicted and measured \gls{pcv} vectors that are consistent with each other. The contact spoke is therefore expected to have a lower residual and \gls{md} value than the other spokes. The contact estimation is explained in detail in Section~\ref{subsec:mahala-based-contact-estimation}. 
    
    In the implemented filter update, the \gls{lf} method is used to select the contact spokes. The \gls{md} is then used to reject inconsistent spoke measurements before the update. In addition, the \gls{md} values based contact estimation is a possible alternative to the \gls{lf} method, as explained in Section~\ref{subsec:mahala-based-contact-estimation}.
    
    \section{Invariant EKF based Odometry}
    
    The \gls{inekf} is an extension of the Extended Kalman Filter whose state evolve on Lie groups. In the \gls{inekf}, the estimation error is defined as an invariant error formulation, such as left or right invariant error rather than Euclidean vector subtraction. The details on the \gls{inekf} and the invariant error are explained in Chapter~\ref{chap:background}. The proposed approach for odometry estimation in \gls{rwr} uses the proprioceptive sensors available on the rover. The \gls{imu} and the joint encoders are used in the fusion process. Kalman filters operate in two steps: the propagation step and the correction step. In this thesis, we use the \gls{imu} data for the propagation step and the joint encoder data is used to construct the measurement used in the correction step. The \gls{inekf} estimates the pose of the rover's body frame with respect to the $odom$ frame, the velocity of the rover and the \gls{imu} biases. The filter's state vector and the details on the propagation step are provided in the following subsections.

    \subsection{Filter State Vector}    
    The state vector $X$ contains the quantities that need to be estimated by the filter. In this work, the state vector contains position, orientation, velocity and \gls{imu} sensor biases. These quantities are required to estimate the rover motion. The filter state is constructed as shown in the Equation~\ref{eq:stateVector}. 
    
    \begin{equation}
        X = [R, V, P, b_g, b_a]
        \label{eq:stateVector}
    \end{equation}
    where, $R$ is the rotation matrix, that represents the orientation of the body frame with respect to the odom frame. $V$ is the velocity vector of the body frame in the odom frame, $P$ is the position vector of the body frame in the odom frame, $b_g$ is the gyroscope bias, $b_a$ is the accelerometer bias. 
    
    \subsection{Propagation step}
    
    The propagation step propagates the current filter state defined in Equation~\ref{eq:stateVector}, to the next time step using the inertial motion model and the \gls{imu} data to generate the State vector at $k+1$ time. The state covariance values are propagated using the state transition matrix and the process noise matrix. This thesis does not modify the propagation step and uses the original implementation \cite{REPO-inekf-github}. The approach presented by Hartley et al. \cite{Hartley2018ContactAidedInEKF}, the authors of the original \gls{inekf} implementation augment the state by adding the position of contact points to the state vector. In contrast, this thesis keeps the state vector unchanged and only estimates the state variables mentioned in Equation~\ref{eq:stateVector}.
    
    \section{Measurement Update Step}
    
    This section presents the parts of the proposed approach that are adapted for the \gls{rwr} system, mainly the \gls{pcv}-based measurement update and its use in the \gls{inekf} correction step. For the fusion process, the \gls{pcv} vector is calculated. This is the velocity of a wheel spoke tip expressed in the body frame in a \gls{rwr} system. The \gls{pcv} can also be referred to as spoke-tip velocity or contact velocity. The term "Potential" comes from the possibility that any one of the spokes in a wheel can be in contact depending on the terrain. The \gls{pcv} is calculated in two forms. The \gls{pcv} vector, calculated from the filter state vector, after the prediction step of the \gls{inekf} is called predicted \gls{pcv}. In the update step, the \gls{pcv} vector is calculated from the joint encoder measurements called measured \gls{pcv}. Within the \gls{inekf}, the predicted and measured \gls{pcv} vectors are used to calculate the residual, also known as the innovation.
    
    \subsection{Predicted Potential Contact Velocity} \label{subsec:predictedPCV}
    
    The predicted \gls{pcv} is calculated after the propagation step of the \gls{inekf}. The vector quantities used in the predicted \gls{pcv} calculation are expressed in the body frame of the rover. This is done to ensure the consistency between predicted and measured \gls{pcv}. The velocity of a \gls{pc} fixed to the rover can be calculated with Equation~\ref{eq:velocityOfaPointInRigidBody}. The rigid body equation is negated to obtain the predicted \gls{pcv} as shown in Equation~\ref{eq:measurementFunction}.
        
    \begin{equation}
        v_{pc,rigid}^{body} = ([R_{body}^{odom}]^T  v_{body}^{odom} + (\omega_{body}^{body} \times r_{pc}^{body}))
        \label{eq:velocityOfaPointInRigidBody}
    \end{equation}
    
    \begin{equation}
        v_{pc, predicted}^{body} = -([R_{body}^{odom}]^T  v_{body}^{odom} + (\omega_{body}^{body} \times r_{pc}^{body}))
        \label{eq:measurementFunction}
    \end{equation}
    
    where, $R_{body}^{odom}$ is the orientation of the body frame expressed in the odom frame and $[R_{body}^{odom}]^T$ transforms the $v_{body}^{odom}$ to the $v_{body}^{body}$. The $R_{body}^{odom}$ and the $v_{body}^{odom}$ are part of the filter's state vector shown in Equation~\ref{eq:stateVector}. $v_{body}^{odom}$ represents the linear velocity of the body frame expressed in the odom frame. $\omega_{body}^{body}$ is the angular velocity of the body expressed in the body frame. $r_{pc}^{body}$, is the position vector of the \gls{pc} from the body frame of the rover expressed in the body frame.
   
    The $v_{pc, rigid}^{body}$ is the rigid-body velocity of the potential contact point at the spoke tip, expressed in the body frame. This quantity is calculated in Equation~\ref{eq:velocityOfaPointInRigidBody}. The negated form represents the relative velocity of the contact surface with respect to the rover body at this potential contact point. This quantity is used as the predicted \gls{pcv}, or measurement function $h(x)$, as shown in Equation~\ref{eq:measurementFunction}.
    
    To understand this negation, we should look at an example. Imagine we have rover that is moving forward with a velocity of $+2 m/s$, for the sake of simplicity, our imaginary rover does not have rotational velocity. According to rigid body kinematics, the velocity of a point attached to the rover will also be $+2 m/s$. From the rover's body frame of reference, the forward motion of the body will be seen as the ground moving backwards under the rover. Mathematically, this can be represented by the negation of Equation~\ref{eq:velocityOfaPointInRigidBody}, resulting in the velocity of the ground moving under the point of contact to be $-2 m/s$. This perspective is used to compare the predicted and the measured \gls{pcv}
        
    
    \subsection{Measured Potential Contact Velocity} \label{subsec:measuredPCV}
    
    The measured \gls{pcv} is calculated using the wheel's angular velocity expressed as a vector and the position vector of \gls{pc}. The position vector is calculated from the wheel center to the tip of the spoke. The measured \gls{pcv} vector is perpendicular to the position vector and points in the direction of rotation of the wheel. The calculation of the measured \gls{pcv} is shown in Equation~\ref{eq:MeasuredPCVCalculation}. The measured \gls{pcv} vector is transformed to the body frame to be consistent with the predicted \gls{pcv} and before the residual calculation.
    
    \begin{equation}
        v_{pc, measured}^{body} = R_{wheel}^{body} (\omega_{wheel}^{wheel} \times {}^{}r_{pc}^{wheel})
        \label{eq:MeasuredPCVCalculation}
    \end{equation}

    From the rover's body frame of reference, $v_{pc, measured}^{body}$ represents the velocity of the ground moving under the point of contact. This is the same quantity as the $v_{pc, predicted}^{body}$, so the difference between them can be calculated as the residual. The $\omega_{wheel}^{wheel}$ is the angular velocity of the wheel expressed in the wheel frame. The $r_{pc}^{wheel}$ is the position vector of the \gls{pc} from the wheel center expressed in the wheel frame. $R_{wheel}^{body}$ is the orientation of the wheel frame expressed in the body frame.
      
    \subsection{Residual Calculation} \label{subsec:residual}
    
    The residual, also referred to as the innovation, is calculated as the difference between the measured and the predicted \gls{pcv}. Both quantities are expressed in the body frame of reference before comparison. The residual is used in the \gls{inekf} to update the state vector.
    
    \begin{equation}
        r_{pcv} =  {v}_{pc, measured}^{body} - {v}_{pc, predicted}^{body} 
    \label{eq:pcvResidual}
    \end{equation}
    
    If the $r_{pcv}$ is small, the wheel encoder measurements are consistent with the filter's predicted rover motion. If the residual is large, then the wheel encoder measurements are inconsistent with the filter's prediction. This could mean wheel encoder measurements may be affected by errors such as wheel slip, an inconsistent contact assumption or noise in wheel encoder measurements. The residual is also used in the \gls{md} calculation which is later used in slip rejection and contact estimation.

    \section{Measurement Model and Jacobian} \label{sec:measurementModelAndJacobian}
        
    The measurement model, also referred to as the measurement function $h(x)$, is given by Equation~\ref{eq:measurementFunction}. The $h(x)$ corresponds to the predicted \gls{pcv} which is used by \gls{inekf} to calculate the predicted measurement from its state vector that can be compared to a measured \gls{pcv} derived from the wheel encoder measurements. The $h(x)$ is also used to derive the measurement Jacobian. For better readability, the measurement function is rewritten in Equation~\ref{eq:measurementFunction-2}. The $\omega_{body}^{body}$ is the angular velocity of the body frame expressed in the body frame with the gyroscope bias removed. This is shown in Equation~\ref{eq:gyroBiasRemoved}.    
    The measurement Jacobian $H$ is the partial derivative of the measurement function $h(x)$ with respect to the error state $\xi$ variables evaluated at the current state $\hat{X}$ as shown in Equation~\ref{eq:MeasurementJacobian-1}. The measurement Jacobian linearizes the measurement function at the current state. The measurement Jacobian describes the relationship between a small perturbation in the state vector and the predicted \gls{pcv} value. The measurement Jacobian calculation and its results for this measurement function is shown in equations \ref{eq:MeasurementJacobian-2} and \ref{eq:MeasurementJacobian-3} respectively.
        
        \begin{equation}
            h(x) = -([R_{body}^{odom}]^T \cdot v_{body}^{odom} + (\omega_{body}^{body} \times r_{pc}^{body}))
            \label{eq:measurementFunction-2}
        \end{equation}
        
        \begin{equation}
            \omega_{body}^{body} = \omega_{meas}^{body} - b_g
            \label{eq:gyroBiasRemoved}
        \end{equation}

        \begin{equation}
            \xi = \begin{bmatrix}
            \delta R \\
            \delta V \\
            \delta P \\
            \delta gyro\_bias \\
            \delta acc\_bias
            \end{bmatrix}
            \label{eq:ErrorStateVector}
        \end{equation}
        
        \begin{equation}
            H = \frac{\partial h(x)}{\partial \xi}\bigg|_{\hat{X}}
            \label{eq:MeasurementJacobian-1}
        \end{equation}
        
        where, $\xi$ is the left invariant error state, refer to Section~\ref{subsec:background_invariant_errors} for details, $\hat{X}$ is the current state estimate, and $h(x)$ is the measurement function. The evaluation of Equation~\ref{eq:MeasurementJacobian-2} to get the result in Equation~\ref{eq:MeasurementJacobian-3} is discussed in detail in the appendix \ref{appendix:MeasurementJacobianDerivation}.
        
        \begin{equation}
            H = \begin{bmatrix}
            \frac{\partial h(x)}{\partial \delta R} & \frac{\partial h(x)}{\partial \delta V} & \frac{\partial h(x)}{\partial \delta P} & \frac{\partial h(x)}{\partial \delta gyro\_bias} & \frac{\partial h(x)}{\partial \delta acc\_bias}
            \end{bmatrix}
            \label{eq:MeasurementJacobian-2}
        \end{equation}
        
        \begin{equation}
            H = \begin{bmatrix}
                -[[R_{body}^{odom}]^T \cdot v_{body}^{odom}]_\times & -\mathbf{I} & \mathbf{0} & -[r_{pc}^{body}]_\times & \mathbf{0}
            \end{bmatrix}
            \label{eq:MeasurementJacobian-3}      
        \end{equation}
        
        The validity of the measurement Jacobian is verified by comparing the analytical Jacobian with a numerical Jacobian. The numerical Jacobian is calculated by perturbing the state vector and calculating the change in the measurement function. The analytical and numerical Jacobians are found to be consistent with each other. The numerical Jacobian calculation is provided in the appendix \ref{appendix:NumericalJacobianCalculation}.

        \section{Slip-aware Measurement Rejection} \label{sec:sliprejection}
        
        The measured \gls{pcv}, calculated from the joint encoder measurements, can be affected by wheel slip. This wheel slip can occur due to several factors such as slippery terrain, compliant terrain like soft sand, loose and uneven terrain like pebbles, inclined terrain, very high wheel speeds or a combination of these factors. The nature of slip and the details required to understand the wheel slip is discussed in Chapter~\ref{chap:background}. 
        
        In the \gls{inekf} framework, the slip-induced inconsistencies may be identified when the measured \gls{pcv} vector calculated from the joint encoders is inconsistent with the predicted \gls{pcv} computed from the filter state. This inconsistency can be calculated using the squared \gls{md}. 
        
        The squared \gls{md} measures residual between a measurement and its predicted values weighted by the inverse innovation covariance \cite{SITE-mathworks-mahalanobis}. The residual calculated in Section~\ref{subsec:residual} is used to calculate the squared \gls{md}. The innovation covariance $S$ is calculated using the Equation~\ref{eq:innovationCovariance} and the squared \gls{md} is calculated using Equation~\ref{eq:mahalanobisDistanceSquared}.
        
        When the squared \gls{md} is higher than a threshold value (details are provided in the Implementation Chapter~\ref{chap:implementation}) then the measured \gls{pcv} is rejected from the update step, thereby preventing the inconsistent joint encoder measurements from a slip event \cite{chang2014MahalanobisFormula}. 
        % \todo[inline]{Include the chi2 threshold in the implementation chapter}

        \begin{equation}
        S = HPH^T + R
        \label{eq:innovationCovariance}
        \end{equation}
        
        \begin{equation}
        d_M^2 = r^T S^{-1} r
        \label{eq:mahalanobisDistanceSquared}
        \end{equation}
        
        where $S$ is the residual or innovation covariance, $H$ is the measurement Jacobian calculated in the previous section, $P$ is the state covariance estimated by the filter, $R$ is the measurement noise covariance, and $r$ is the residual.
        
        \section{Contact Estimation} \label{sec:ContactEstimation}
        
        The residual consistency measure used in the slip-aware measurement rejection is also used to estimate which spoke of a wheel is in contact. The rimless wheel motion can be described as a combination of the swing phase and the contact phase. 
        
        Estimating which spoke tip is currently in contact for a given wheel is important for the Contact Velocity Odometry estimation. 
        
        Correctly identifying the contact spoke is important because the \gls{pcv} residual is calculated for a specific spoke tip. Selecting the wrong spoke can make a valid wheel measurement appear inconsistent, leading to an incorrect measurement rejection.
        
        The contact estimation is performed using two methods, the lowest-foot method and the \gls{md}-based method. 
        
        \subsection{Lowest-Foot-based Contact Estimation}
        
        A geometrical method of estimating the contact spokes is by using the assumption that the spoke tip that has the lowest vertical position is in contact. This can be estimated using the robot kinematics and the joint positions of the wheel. The pose of each spoke with respect to the body frame can be calculated and the spoke in contact can be estimated. The \gls{rw} is constructed with five spokes so, geometrically there could be either one spoke in contact or two. The lowest-foot assumption can estimate only one foot as contact. For multi contact spoke estimation, we assumed a region of vertical position, if the spoke tip is within this region, then it is estimated as in contact. This assumption works well on a flat, solid terrain but on an inclined or rough terrain with obstacles, this assumption may not hold.
              
        \subsection{Mahalanobis-Distance-based Contact Estimation} \label{subsec:mahala-based-contact-estimation}
        
        The use of \gls{inekf} for odometry estimation provides an opportunity to estimate the spokes that are in contact. With the squared \gls{md} calculated in the previous section, the contact spokes can be estimated by calculating the squared \gls{md} value for each spoke and selecting the spoke with the least squared \gls{md} as the spoke in contact. This is based on the assumption that the spoke tip in contact with the ground produces the smallest difference between the predicted and measured \gls{pcv}. Therefore, the spoke with the lowest squared \gls{md} value is selected as the most likely contact spoke. This method can also be used to estimate multiple contact spokes by selecting the spokes with squared \gls{md} value below a certain threshold. The implementation details related to the approach discussed in this chapter is presented in the Implementation Chapter~\ref{chap:implementation}.
