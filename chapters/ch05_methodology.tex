    \chapter{Methodology} \label{chap:methodology}
    
    This chapter provides details on the proposed approach to the odometry estimation problem mentioned in the problem statement Chapter~\ref{chap:problem_statement}. The details of \gls{inekf}-based odometry, the proposed measurement update step for the \gls{inekf}, the Mahalanobis-distance-based measurement rejection, the contact spoke estimation, and the limitations of the proposed approach are presented in this chapter. The overview of the approach is explained in Section~\ref{sec:approach_overview}.

    The proposed approach uses IMU data to propagate the filter state and wheel encoder data to construct the measured PCV. This section explains the intuition behind the proposed approach. The predicted \gls{pcv} vector is calculated for every spoke of each wheel. To simplify the explanation, only one wheel and its spokes are considered here.
    
    Let us take a scenario where the rover moves forward while rotating slightly on its yaw (vertical) axis, so the trajectory from the top-view looks like a part of a circle. During this motion, we need to estimate the position and orientation of the rover in the world frame. The origin of the world frame, also referred to as the odometry frame, usually lies at the location where the rover starts its motion.
    
    The \gls{inekf} estimates the pose of the rover and other elements in its state vector \ref{eq:stateVector}. To estimate these elements, the filter uses the \gls{imu} data to propagate the state vector elements forward in time. Let us define this propagated state $\hat{X}^{-}_{k+1}$ at the time step $k+1$. Let the current state be $X_{k}$ and the current time step be $k$. This propagation of state is done using an inertial motion model driven by the \gls{imu} measurement. This motion model uses the bias corrected gyroscope and accelerometer values to update the orientation, position and velocity of the rover. Since, this predicted state is calculated only with \gls{imu} driven motion model, it accumulates error due to sensor drift. This error needs to be corrected by an independent measurement source. In the proposed approach the correction is done using the wheel encoder measurements.
    
    Using the predicted state $\hat{X}^{-}_{k+1}$, the velocity of each spoke tip can be calculated in the body frame. The details of the predicted \gls{pcv} are provided in Section~\ref{subsec:predictedPCV}.
    To understand the predicted \gls{pcv}, we calculate the velocity that each spoke tip would have due to the predicted body linear velocity, the bias-corrected body angular velocity from \gls{imu}, and the spoke-tip position relative to the body frame. This calculation generates a velocity vector for each spoke tip of the selected wheel, shown as red arrows in Figure~\ref{fig:approachPCVexplanation}.  \todo[inline]{Modify the description of velocity vectors when the image is updated}
    
    To correct the predicted state $\hat{X}^{-}_{k+1}$, the measured \gls{pcv} is calculated using the wheel encoder measurements. The details of the measured \gls{pcv} are provided in Section~\ref{subsec:measuredPCV}. Similar to the predicted \gls{pcv}, we calculate the measured \gls{pcv} for all the spokes of a wheel. The measured \gls{pcv} is calculated from the wheel angular velocity about its rotation axis and the position vector from the wheel center to the spoke tip. The resulting measured \gls{pcv} points along the direction of wheel rotation and is perpendicular to the position vector from the wheel center to the spoke tip. These vectors are shown as the green arrow in Figure~\ref{fig:approachPCVexplanation}. This measured \gls{pcv} is initially expressed in the wheel frame and later transformed into the body frame.
        
    Once both the predicted and measured \gls{pcv} are calculated, the spokes assumed to be in contact need to be selected and used to update the state vector. This can be done in two ways as explained in Section~\ref{sec:ContactEstimation}. The proposed approach uses the \gls{lf} method to select the spokes that are used in the filter update step. These selected spoke are used to calculate the residual as discussed in Section~\ref{subsec:residual}. After the residual is calculated, the covariance of the residual is calculated using Equation~\ref{eq:innovationCovariance}. This covariance is then used to calculate the squared Mahalanobis distance. This distance represents how surprised the filter is to receive the measured \gls{pcv}. 
    
    From this information, it can be statistically decided whether a spoke measurement is consistent with the predicted state. If the squared \gls{md} is above the chi-squared threshold, the corresponding spoke measurement is considered inconsistent, which may indicate wheel slip or incorrect contact assumption and is rejected. This spoke is rejected from the update step. Only spokes, with squared \gls{md} within the threshold are used to update the filter state. After the update, the filter produces the corrected state $\hat{X}^{+}_{k+1}$. 
    
    The calculated \gls{md} is also used to estimate the spokes that are likely to be in contact. The spoke with the lowest \gls{md} is selected to be most likely in contact. This can be seen in Figure~\ref{fig:approachPCVexplanation}, where the spoke estimated to be in contact with the ground has the predicted and measured \gls{pcv} vectors that are consistent with each other. This is reflected by a low residual and a low \gls{md} value. The other spokes that have inconsistent predicted and measured \gls{pcv} vectors have higher residuals and higher \gls{md} values. The contact estimation is explained in detail in Section~\ref{subsec:mahala-based-contact-estimation}. 
    
    In the implemented filter update, the \gls{lf} method is used to select the contact spokes. The \gls{md} is then used to reject inconsistent spoke measurements before the update. In addition, the \gls{md} values can also be used as an alternative contact estimation measure, as explained in Section~\ref{subsec:mahala-based-contact-estimation}.
    
    
    
    \section{Invariant EKF based Odometry}
    
    The \gls{inekf} is an extension of the Extended Kalman Filter whose state evolve on Lie groups. In the \gls{inekf}, the estimation error is defined as an invariant error formulation, such as left or right invariant error rather than Euclidean vector subtraction. The details on the \gls{inekf} and the invariant error are explained in Chapter~\ref{chap:background}. 
    
    % This makes the estimation error independent of the frame of reference. The invariance property of the \gls{inekf} is explained in Chapter~\ref{chap:background}. 
        
    % The advantage of using \gls{inekf} over standard EKF is that the \gls{inekf} has better convergence properties.

    The proposed approach for odometry estimation in \gls{rwr} uses the proprioceptive sensors available on the rover. The \gls{imu} and the wheel encoders are used in the fusion process. Kalman filters operate in two steps: the propagation step and the correction step. In this thesis, we use the \gls{imu} data for the propagation step and the wheel encoder data is used to construct the measurement used in the correction step. The \gls{inekf} estimates the pose of the rover's body frame with respect to the world frame, the velocity of the rover and the \gls{imu} biases. The filter's state vector and the details on the propagation step are provided in the following subsections.

    \subsection{Filter State Vector}    
    The state vector $X$ contains the quantities that need to be estimated by the filter. In this work, the state vector contains position, orientation, velocity and \gls{imu} sensor biases. These quantities are required to estimate the rover motion. The filter state is constructed as shown in the Equation~\ref{eq:stateVector}. 
    
    \begin{equation}
        X = [R, V, P, b_g, b_a]
        \label{eq:stateVector}
    \end{equation}
    where, $R$ is the rotation matrix, that represents the orientation of the body frame with respect to the world frame. $V$ is the velocity vector of the body frame in the world frame, $P$ is the position vector of the body frame in the world frame, $b_g$ is the gyroscope bias, $b_a$ is the accelerometer bias. 
    
    \subsection{Propagation step}
    
    The propagation step propagates the current filter state defined in Equation~\ref{eq:stateVector}, to the next time step using the inertial motion model and the \gls{imu} data. The state covariance values are propagated using the state transition matrix and the process noise matrix. This thesis does not modify the propagation step and uses the original implementation \cite{REPO-inekf-github}. The approach presented by Hartley et al. \cite{Hartley2018ContactAidedInEKF}, the authors of the original \gls{inekf} implementation augment the state by adding the position of contact points to the state vector. In contrast, this thesis keeps the state vector unchanged and only estimates the state variables mentioned in Equation~\ref{eq:stateVector}.
    
    \section{Measurement Update Step}
    
    This section presents the core methodology of the proposed approach. For the fusion process, the \gls{pcv} vector is calculated. This is the velocity of a wheel spoke tip expressed in the body frame in a \gls{rwr} system. The \gls{pcv} can also be referred to as spoke-tip velocity or contact velocity. The term "Potential" comes from the possibility that any one of the spokes in a wheel can be in contact depending on the terrain. The \gls{pcv} is calculated in two forms. The \gls{pcv} vector, calculated from the filter state vector, after the prediction step of the \gls{inekf} is called predicted \gls{pcv}. In the update step, the \gls{pcv} vector is calculated from the wheel encoder measurements called measured \gls{pcv}. Within the \gls{inekf}, the predicted and measured \gls{pcv} vectors are used to calculate the residual, also known as the innovation.
    
    \subsection{Predicted Potential Contact Velocity} \label{subsec:predictedPCV}
    
    The predicted \gls{pcv} is calculated after the propagation step of the \gls{inekf}. The vector quantities used in the predicted \gls{pcv} calculation are expressed in the body frame of the rover. This is done to ensure the consistency between predicted and measured \gls{pcv}.
    The velocity of a \gls{pc} fixed to the rover can be calculated with Equation~\ref{eq:velocityOfaPointInRigidBody}. The rigid body equation is negated to obtain the predicted \gls{pcv} as shown in Equation~\ref{eq:measurementFunction}.
        
    \begin{equation}
        v_{pc,rigid}^{body} = ([R_{body}^{world}]^T  v_{body}^{world} + (\omega_{body}^{body} \times r_{pc}^{body}))
        \label{eq:velocityOfaPointInRigidBody}
    \end{equation}
    
    \begin{equation}
        v_{pc, predicted}^{body} = -([R_{body}^{world}]^T  v_{body}^{world} + (\omega_{body}^{body} \times r_{pc}^{body}))
        \label{eq:measurementFunction}
    \end{equation}
    
    where $r_{pc}^{body}$, is the position vector of the \gls{pc} from the body frame of the rover expressed in the body frame, $\omega_{body}^{body}$ is the angular velocity of the body expressed in the body frame. $v_{body}^{world}$ represents the linear velocity of the body frame expressed in the world frame. $R_{body}^{world}$ is the orientation of the body frame expressed in the world frame and $[R_{body}^{world}]^T$ transforms the $v_{body}^{world}$ to the $v_{body}^{body}$. The $R_{body}^{world}$ and the $v_{body}^{world}$ are part of the filter's state vector \ref{eq:stateVector}.
    
    The $v_{pc, rigid}^{body}$ is the velocity of a point on the rover moving through the world expressed in the body frame. This quantity is calculated in Equation~\ref{eq:velocityOfaPointInRigidBody}. The negation of this equation represents the velocity with which the world is moving under the rover, also referred to as the measurement function $h(x)$. This quantity is calculated using the Equation~\ref{eq:measurementFunction}. 
    
    To understand this negation, we should look at an example. Imagine we have rover that is moving forward with a velocity of $+2 m/s$, for the sake of simplicity, our imaginary rover does not have rotational velocity. According to rigid body kinematics, the velocity of a point attached to the rover will also be $+2 m/s$. From the rover's body frame of reference, the forward motion of the body will be seen as the ground moving backwards under the rover. Mathematically, this can be represented by the negation of Equation~\ref{eq:velocityOfaPointInRigidBody}, resulting in the velocity of the ground moving under the point of contact to be $-2 m/s$. This shift in perspective is used to compare the predicted and the measured \gls{pcv}
        
    
    \subsection{Measured Potential Contact Velocity} \label{subsec:measuredPCV}
    
    The measured \gls{pcv} is calculated using the wheel's angular velocity expressed as a vector and the position vector of \gls{pc}. The position vector is calculated from the wheel center to the tip of the spoke. The measured \gls{pcv} vector is perpendicular to the position vector and points in the direction of rotation of the wheel. The calculation of the measured \gls{pcv} is shown in Equation~\ref{eq:MeasuredPCVCalculation}. The measured \gls{pcv} vector is transformed to the body frame to be consistent with the predicted \gls{pcv} and before the residual calculation.
    
    \begin{equation}
        v_{pc, measured}^{body} = R_{wheel}^{body} (\omega_{wheel}^{wheel} \times {}^{}r_{pc}^{wheel})
        \label{eq:MeasuredPCVCalculation}
    \end{equation}
    

    From the rover's body frame of reference, $v_{pc, measured}^{body}$ represents the velocity of the ground moving under the point of contact. This is the same quantity as the $v_{pc, predicted}^{body}$, so the difference between them can be calculated as the residual. The $\omega_{wheel}^{wheel}$ is the angular velocity of the wheel expressed in the wheel frame. The $r_{pc}^{wheel}$ is the position vector of the \gls{pc} from the wheel center expressed in the wheel frame. $R_{wheel}^{body}$ is the orientation of the wheel frame expressed in the body frame.
      
    \subsection{Residual Calculation} \label{subsec:residual}
    
    The residual, also referred to as the innovation, is calculated as the difference between the measured and the predicted \gls{pcv}. Both quantities are expressed in the body frame of reference before comparison. The residual is used in the \gls{inekf} to update the state vector. The implementation details are provided in Chapter~\ref{chap:implementation}.
    
    \begin{equation}
        r_{pcv} =  {v}_{pc, measured}^{body} - {v}_{pc, predicted}^{body} 
    \label{eq:pcvResidual}
    \end{equation}
    
    If the $r_{pcv}$ is small, the wheel measurements are consistent with the filter's predicted rover motion. If the residual is large, then the wheel measurements are inconsistent with the filter's prediction. This could mean wheel measurements may be affected by errors such as wheel slip, an inconsistent contact assumption or noise in wheel measurements. The residual is also used in the \gls{md} calculation which is later used in slip rejection and contact estimation.

    \section{Measurement Model and Jacobian}
        
    The measurement model, also referred to as the measurement function $h(x)$, is given by Equation~\ref{eq:measurementFunction}. The $h(x)$ corresponds to the predicted \gls{pcv} which is used by \gls{inekf} to calculate the predicted measurement from its state vector that can be compared to a measured \gls{pcv} derived from the wheel measurements. The $h(x)$ is also used to derive the measurement Jacobian. For better readability, the measurement function is rewritten in Equation~\ref{eq:measurementFunction-2}. The $\omega_{body}^{body}$ is the angular velocity of the body frame expressed in the body frame with the gyroscope bias removed. This is shown in Equation~\ref{eq:gyroBiasRemoved}.    
    The measurement Jacobian $H$ is the partial derivative of the measurement function $h(x)$ with respect to the error state $\xi$ variables evaluated at the current state $\hat{X}$ as shown in Equation~\ref{eq:MeasurementJacobian-1}. The measurement Jacobian linearizes the measurement function at the current state. The measurement Jacobian describes the relationship between a small perturbation in the state vector and the predicted \gls{pcv} value. The measurement Jacobian calculation and its results for this measurement function is shown in equations \ref{eq:MeasurementJacobian-2} and \ref{eq:MeasurementJacobian-3} respectively.
        
        \begin{equation}
            h(x) = -([R_{body}^{world}]^T \cdot v_{body}^{world} + (\omega_{body}^{body} \times r_{pc}^{body}))
            \label{eq:measurementFunction-2}
        \end{equation}
        
        \begin{equation}
            \omega_{body}^{body} = \omega_{meas}^{body} - b_g
            \label{eq:gyroBiasRemoved}
        \end{equation}

    
        \begin{equation}
            \xi = \begin{bmatrix}
            \delta R \\
            \delta V \\
            \delta P \\
            \delta acc\_bias \\
            \delta gyro\_bias
            \end{bmatrix}
            \label{eq:ErrorStateVector}
        \end{equation}
        
        \begin{equation}
            H = \frac{\partial h(x)}{\partial \xi}\bigg|_{\hat{X}}
            \label{eq:MeasurementJacobian-1}
        \end{equation}
        
        where, $\xi$ is the left invariant error state, refer to Section~\ref{sec:LeftAndRightInvariantError} for details, $\hat{X}$ is the current state estimate, and $h(x)$ is the measurement function. The evaluation of Equation~\ref{eq:MeasurementJacobian-2} to get the result in Equation~\ref{eq:MeasurementJacobian-3} is discussed in detail in the appendix \ref{appendix:MeasurementJacobianDerivation}.
        
        \begin{equation}
            H = \begin{bmatrix}
            \frac{\partial h(x)}{\partial \delta R} & \frac{\partial h(x)}{\partial \delta V} & \frac{\partial h(x)}{\partial \delta P} & \frac{\partial h(x)}{\partial \delta acc\_bias} & \frac{\partial h(x)}{\partial \delta gyro\_bias}
            \end{bmatrix}
            \label{eq:MeasurementJacobian-2}
        \end{equation}
        
        \begin{equation}
            H = \begin{bmatrix}
                -[[R_{body}^{world}]^T \cdot v_{body}^{world}]_\times & -\mathbf{I} & \mathbf{0} & \mathbf{0} & -[r_{pc}^{body}]_\times
            \end{bmatrix}
            \label{eq:MeasurementJacobian-3}      
        \end{equation}
        
        The validity of the measurement Jacobian is verified by comparing the analytical Jacobian with a numerical Jacobian. The numerical Jacobian is calculated by perturbing the state vector and calculating the change in the measurement function. The analytical and numerical Jacobians are found to be consistent with each other. The numerical jacobian calculation is provided in the appendix \ref{appendix:NumericalJacobianCalculation}.

        \section{Slip-aware Measurement Rejection}
        
        The measured \gls{pcv}, calculated from the wheel encoder measurements, can be affected by wheel slip. This wheel slip can occur due to several factors such as slippery terrain, compliant terrain like soft sand, loose and uneven terrain like pebbles, inclined terrain, very high wheel speeds or a combination of these factors. The nature of slip and the details required to understand the wheel slip is discussed in Chapter~\ref{chap:background}. 
        
        In the \gls{inekf} framework, the slip-induced inconsistencies may be identified when the measured \gls{pcv} vector calculated from the wheel encoders is inconsistent with the predicted \gls{pcv} computed from the filter state. This inconsistency can be calculated using the \gls{md}. 
        
        The \gls{md} measures residual between a measurement and its predicted values weighted by the inverse residual covariance \cite{SITE-mathworks-mahalanobis}. The residual calculated in Section~\ref{subsec:residual} is used to calculate the \gls{md}. When the squared \gls{md} is higher than a threshold value (details are provided in the Implementation Chapter~\ref{chap:implementation})\todo[inline]{Include the chi2 threshold in the implementation chapter} then the measured \gls{pcv} is rejected from the update step, thereby preventing the inconsistent wheel encoder measurements from a slip event \cite{chang2014MahalanobisFormula}.
        
        \begin{equation}
        S = HPH^T + R
        \label{eq:innovationCovariance}
        \end{equation}
        
        \begin{equation}
        d_M^2 = r^T S^{-1} r
        \label{eq:mahalanobisDistanceSquared}
        \end{equation}
        
        where $S$ is the residual or innovation covariance, $H$ is the measurement Jacobian calculated in the previous section, $P$ is the state covariance estimated by the filter, $R$ is the measurement noise covariance, and $r$ is the residual.
        
        \section{Contact Estimation} \label{sec:ContactEstimation}
        
        The residual consistency measure used in the slip-aware measurement rejection is also used to estimate which spoke of a wheel is in contact. Since the rimless wheel design is a hybrid between wheeled and legged systems, its motion can be described as a combination of the swing phase and the contact phase. Estimating which spoke tip is currently in contact for a given wheel is important for the Contact Velocity Odometry estimation. The contact estimation is performed using two methods, the lowest-foot method and the \gls{md}-based method. 
        
        \subsection{Lowest-Foot-based Contact Estimation}
        
        A geometrical method of estimating the contact spoke is by using the assumption that the spoke tip that has the lowest vertical position is in contact. This can be estimated using the robot kinematics and the joint positions of the wheel. The pose of each spoke with respect to the body frame can be calculated and the spoke in contact can be estimated. The \gls{rw} is constructed with five spokes so, geometrically there could be either one spoke in contact or two. The lowest-foot assumption can estimate only one foot as contact. For multi contact spoke estimation, we assumed a region of vertical position, if the spoke tip is within this region, then it is estimated as in contact. This assumption works well on a flat, solid terrain but on an inclined or rough terrain with obstacles, this assumption may not hold.
                
              
        \subsection{Mahalanobis-Distance-based Contact Estimation} \label{subsec:mahala-based-contact-estimation}
        
        The use of \gls{inekf} for odometry estimation provides an opportunity to estimate the spokes that are in contact. With the \gls{md} squared calculated in the previous section, the contact spokes can be estimated by calculating the \gls{md} squared value for each spoke and selecting the spoke with the lowest \gls{md} squared as the spoke in contact. This is based on the fact that, the spoke with lowest \gls{md} squared value's predicted and measured \gls{pcv} will be consistent compared to the other spokes with higher \gls{md} squared value. This method can also be used to estimate multiple contact spokes by selecting the spokes with \gls{md} squared value below a certain threshold.
